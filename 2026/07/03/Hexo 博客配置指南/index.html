Hexo 博客配置指南：从默认站点到个性化博客

基于主题：hexo-theme-A4 v2.0.1（其他主题思路通用）

写在前面

安装指南带你把 Hexo 跑起来了——浏览器打开 http://localhost:4000 能看到默认页面。但那只是出厂设置：标题是”Hexo”，副标题空着，邮箱是个占位符，访问量数字离谱地大。

这篇指南要做的事情很简单：把这些默认占位内容全部换成你自己的。从改标题、换头像、调主题样式，到创建标签页、分类页、关于页，再到发布第一篇文章。

打开博客根目录下的 _config.yml，这是 Hexo 的站点配置文件（也叫主配置文件），控制整个网站的全局信息。找到 Site 这一段：

各字段说明

字段 作用 建议
title 博客名称，出现在浏览器标签页和首页最显眼的位置 简短好记，体现个人品牌
subtitle 副标题，通常显示在标题下方或首页头部区域 一句话概括博客定位

💡 改完保存后，运行 hexo clean && hexo g && hexo s 刷新浏览器即可看到变化。不需要重启电脑或重装任何东西，配置文件改动后重新生成就能生效。

四、头像 / 图标资源放置规则

这是 Hexo 新手最容易搞错的地方之一——图片放错了位置导致 404。

4.1 核心规则：只有 source/ 目录下的文件才会被处理

Hexo 在生成静态网站时，只会处理 source/ 文件夹里的内容。放在其他位置的文件（比如博客根目录下随便建个 img 文件夹）完全不会被复制到最终网站里。

七、配置阶段常见问题汇总

以下是我在配置过程中实际遇到过的所有问题，按出现频率排序。

Q1：修改 _config.yml 后报 FATAL YAMLError

现象：命令行弹出黄色错误信息，提示 bad indentation of a mapping entry。

原因：YAML 格式错误，最常见两种情况：

  1. 冒号后面缺少空格（见第二节 规则一）
  2. 单引号嵌套冲突（见第二节 规则二）

解决：逐行检查报错行附近的格式，对照 YAML 两条铁律修正。

Q2：hexo s 输入后只显示帮助信息，没有启动服务器

现象：输入 hexo s 后没有出现 Hexo is running，而是弹出一堆命令列表。

原因：当前目录不是有效的 Hexo 项目目录。常见于：

  • 迁移目录后 node_modules 没有跟着过来
  • cd 命令没有真正切到目标目录（跨盘符忘了加 /d）

现象：服务器启动成功了（能看到 Hexo is running at http://localhost:4000），但浏览器打开页面时报错，命令行显示：

原因：主题模板在渲染时读取到了 null 值。几乎都是因为修改配置文件时某个字段被清空或删除了，导致模板拿不到预期的变量。

解决：

  1. 先恢复默认配置确认是配置问题还是其他原因：

  2. 如果恢复正常了，说明确实是配置问题。然后逐项修改——每次只改一个字段，改完刷新验证，定位是哪个字段导致的。

  3. 以后改配置遵循安全原则：可以改值、可以注释、不要整行删除。

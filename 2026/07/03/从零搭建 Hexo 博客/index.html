写在前面

Hexo 是一个基于 Node.js 的静态博客框架。简单来说，你只需要用 Markdown 写文章，Hexo 就能帮你自动生成漂亮的网页，然后一键部署到 GitHub Pages 或其他平台——免费、快速、不需要自己搭服务器。

本文将手把手带你从零开始，完成环境安装、博客初始化、本地预览的全过程。每一步都配有实际操作截图和常见报错解决方案。

一、准备工作：安装 Node.js 和 Git

Hexo 运行需要两个基础环境：Node.js 和 Git。一个都不能少。

1.1 安装 Node.js

Node.js 是 JavaScript 的运行环境，Hexo 就是用它来跑的。npm（Node Package Manager）是随 Node.js 一起安装的包管理工具，后面我们用 npm install 来装 Hexo 就是靠它。

  • 访问官网下载：https://nodejs.org/
  • 选择 LTS 版本（长期支持版），更稳定
  • 下载后双击安装包，一路”下一步”即可，不需要额外配置环境变量

安装完成后，打开命令行工具验证：

如果能显示版本号（类似 v20.x.x 和 10.x.x），说明安装成功。

Windows 用户提示：打开命令行的方式是按 Win + R，输入 cmd 回车；或者按 Win + X 选择”终端”或”PowerShell”。macOS/Linux 用户直接打开”终端”（Terminal）即可。

1.2 安装 Git

Git 是版本控制工具，Hexo 用它来管理主题文件和部署代码。

  • 访问官网下载：https://git-scm.com/downloads
  • 选择对应操作系统的版本下载安装
  • Windows 用户注意：安装过程中有一个选项 “Use Git from the Windows Command Prompt”，建议勾选上，这样可以直接在 cmd 里使用 git 命令

附录 A：博客目录迁移指南（C 盘 → D 盘）

如果你一开始把博客装在了 C 盘（比如 C:\Users\Jo\myblog），后来想迁移到 D 盘（比如 D:\Jo\myblog），操作非常简单。Hexo 项目本质就是一个普通文件夹，所有路径都是相对路径，不跟固定盘符绑定。

为什么推荐放 D 盘

  • 系统重装不怕丢数据：C 盘是系统盘，重装或格式化时首当其冲；D 盘的数据通常不受影响
  • 不挤占系统空间：node_modules 文件夹通常一两百 MB，加上后续的文章图片，体积只增不减；C 盘太满会导致系统变卡
  • 方便备份管理：D 盘是存放个人数据的常规位置，备份时思路更清晰

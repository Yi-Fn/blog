前言

本地跑起来的博客只有自己能看，部署到线上才算真正”开张”。本文记录了 Hexo 博客部署到 GitHub 的完整过程。部署完成后，你会获得一个 https://你的用户名.github.io/仓库名/ 的公开网址，任何人都能访问。

一、前置准备

在开始部署之前，确保你已经完成以下准备：

  • GitHub 账号：去 github.com 注册，用户名会出现在博客地址里，建议选一个满意的
  • 本地 Hexo 博客已跑通：hexo s 能在 localhost:4000 正常预览
  • Git 已安装：命令行输入 git --version 能显示版本号

二、在 GitHub 上创建仓库

登录 GitHub 后，点击右上角 + 号，选择 New repository。

关键要求：

  • Repository name：可以任意取，比如取名叫 blog
  • 选择 Public（公开），否则 Pages 无法正常访问
  • 不要勾选 “Add a README file”，创建一个空仓库即可

6.1 认证问题

因为用的是 HTTPS 地址，第一次推送时 GitHub 会要求认证。

如 GitHub 不支持用登录密码推送，需要用 Personal Access Token 代替密码。

生成 Token 的方法：

  1. GitHub → 头像 → Settings → 最底部 Developer settings
  2. Personal access tokens → Tokens (classic) → Generate new token (classic)
  3. Note 随便填（比如”blog部署”），Expiration 选时间
  4. 勾选 repo 权限（第一个大项）
  5. 点 Generate token，立刻复制生成的令牌（只显示一次）

部署弹窗输入：

  • 用户名：你的 GitHub 用户名
  • 密码：粘贴刚才的 Token

6.3 网络连接问题

国内访问 GitHub 不稳定，hexo d 时可能遇到 Failed to connect to github.com:443 的报错。解决办法：

  • 如果开了代理工具，给 Git 配置代理：git config --global http.proxy http://127.0.0.1:端口号
  • 或者换个网络环境重试
  • 多试几次，有时候只是临时连不上

七、开启 GitHub Pages

代码推送成功后，还需要在 GitHub 上开启 Pages 服务：

  1. 打开仓库页面，点顶部 Settings
  2. 左侧菜单找到 Pages
  3. Source 选择 Deploy from a branch
  4. 分支选 main，目录选 /(root)
  5. 点 Save

保存后等 1-2 分钟，页面顶部出现绿色提示 Your site is live，就说明网站正式上线了。

十、踩坑记录

部署过程中可能踩的几个坑，记录下来供参考：

坑一：YAML 格式错误

配置文件里冒号后面漏了空格，或者用了 Tab 键缩进，都会导致解析报错。YAML 对格式非常敏感，记住两条铁律：冒号后必须有空格，缩进只用空格不用 Tab。

坑二：资源路径 404

仓库名不是 用户名.github.io 时，忘记设置 root: /仓库名/，导致页面样式全丢。这是项目站点最容易踩的坑。

坑三：Git 身份未配置

新电脑上第一次用 Git，没设置 user.name 和 user.email，部署直接报错。

坑四：GitHub 密码认证失效

用登录密码推送被拒绝，需要改用 Personal Access Token 或 SSH 密钥。

坑五：网络连不上 GitHub

国内网络环境不稳定，hexo d 时连不上 GitHub 服务器。需要配置代理或多次重试。
